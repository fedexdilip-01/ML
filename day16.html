📐 What is ML/AI Architecture Design?

🎯 High-Level Architecture

What: Bird's eye view of system components and their interactions. Data sources, storage, processing, model training, inference, monitoring.

Why: Aligns stakeholders, shows data flow, identifies bottlenecks, guides technology selection.

Where: System design interviews, solution architecture documents, team presentations.

⚙️ Low-Level Architecture

What: Detailed design of each component: APIs, database schemas, class diagrams, algorithm pseudocode, infrastructure specs.

Why: Enables implementation, ensures scalability, defines error handling, specifies interfaces.

Where: Development sprints, code reviews, technical specifications.

# ============ ALGORITHM PSEUDOCODE ============
class TwoTowerRecommender:
    """Low-level implementation of two-tower neural network"""
    
    def __init__(self, embedding_dim=128, learning_rate=0.001):
        self.user_tower = self._build_user_tower(embedding_dim)
        self.item_tower = self._build_item_tower(embedding_dim)
        self.embedding_dim = embedding_dim
    
    def _build_user_tower(self, dim):
        # Input: user_id, user_features (age, purchase_history_summary)
        return tf.keras.Sequential([
            tf.keras.layers.Dense(256, activation='relu'),
            tf.keras.layers.Dense(128, activation='relu'),
            tf.keras.layers.Dense(dim)
        ])
    
    def _build_item_tower(self, dim):
        # Input: product_id, category, price
        return tf.keras.Sequential([
            tf.keras.layers.Dense(256, activation='relu'),
            tf.keras.layers.Dense(128, activation='relu'),
            tf.keras.layers.Dense(dim)
        ])
    
    def predict(self, user_embedding, item_embeddings):
        # Cosine similarity (dot product)
        scores = np.dot(item_embeddings, user_embedding)
        return scores
    
    def train_step(self, user_batch, item_batch, labels):
        # Loss: Binary cross-entropy for positive/negative pairs
        user_emb = self.user_tower(user_batch)
        item_emb = self.item_tower(item_batch)
        scores = tf.reduce_sum(user_emb * item_emb, axis=1)
        loss = tf.reduce_mean(tf.nn.sigmoid_cross_entropy_with_logits(labels, scores))
        return loss

# ============ HIGH-LEVEL (Simplified) ============
"""
Components:
1. Data: ratings.csv, movies.csv (local files)
2. Processing: Pandas in Jupyter Notebook
3. Model: Surprise library (SVD) or LightFM
4. Inference: Flask API (single endpoint)
5. Frontend: Streamlit dashboard
"""

# ============ LOW-LEVEL IMPLEMENTATION ============
import pandas as pd
from surprise import Dataset, Reader, SVD
from surprise.model_selection import train_test_split
import joblib
from flask import Flask, request, jsonify

# ---- Training Script (train.py) ----
def train_movie_recommender():
    # Load data
    ratings = pd.read_csv('ratings.csv')
    reader = Reader(rating_scale=(0.5, 5.0))
    data = Dataset.load_from_df(ratings[['userId', 'movieId', 'rating']], reader)
    trainset, testset = train_test_split(data, test_size=0.2)
    
    # Train SVD model
    model = SVD(n_factors=50, lr_all=0.005, reg_all=0.02)
    model.fit(trainset)
    
    # Save model
    joblib.dump(model, 'movie_recommender.pkl')
    return model

# ---- API (app.py) ----
app = Flask(__name__)
model = joblib.load('movie_recommender.pkl')

@app.route('/predict', methods=['POST'])
def predict():
    data = request.json
    user_id = data['user_id']
    movie_id = data['movie_id']
    prediction = model.predict(user_id, movie_id).est
    return jsonify({'user_id': user_id, 'movie_id': movie_id, 'rating': prediction})

@app.route('/recommend/', methods=['GET'])
def recommend(user_id):
    # Get all movies not seen by user, predict ratings, return top 10
    all_movies = pd.read_csv('movies.csv')['movieId'].tolist()
    predictions = [(movie, model.predict(user_id, movie).est) for movie in all_movies]
    top_10 = sorted(predictions, key=lambda x: x[1], reverse=True)[:10]
    return jsonify({'user_id': user_id, 'recommendations': top_10})

if __name__ == '__main__':
    app.run(port=5000)

# ---- Streamlit Frontend (ui.py) ----
import streamlit as st
import requests

st.title("🎬 Movie Recommender")
user_id = st.text_input("Enter User ID", "1")
if st.button("Get Recommendations"):
    response = requests.get(f"http://localhost:5000/recommend/{user_id}")
    recs = response.json()['recommendations']
    for movie_id, rating in recs:
        st.write(f"Movie {movie_id}: predicted rating {rating:.2f}")

✓ What data sources are available?
✓ What's the prediction latency requirement?

✓ How often to retrain?
✓ Expected QPS?
✓ Cloud or on-prem?

🌟 Complete ML/AI Architecture Summary:

High-Level Architecture answers "What components exist and how do they interact?" — focuses on data flow, system boundaries, technology choices, and stakeholder alignment.

Low-Level Architecture answers "How exactly does each component work?" — includes API specs, database schemas, algorithm pseudocode, class diagrams, and infrastructure configuration.

Example (RecSys): High-level: Data Sources → ETL → Training → Registry → Serving → Client. Low-level: Flask endpoints, PostgreSQL VECTOR extension, Two-Tower NN in TensorFlow, Redis TTL policies, Prometheus alerts.

Pro Tip: Always start with high-level to validate requirements, then drill into low-level for critical components. Document both for your team!
