🚀 1. XGBoost – The Power of Boosting
🌳 Gradient Boosting with Regularization & Parallel Processing
📐 Boosting concept

Sequentially trains weak learners (trees), each correcting previous errors. XGBoost adds L1/L2 regularization, tree pruning, and handles missing values.

🎯 Why XGBoost excels

Built-in cross-validation, early stopping, feature importance, GPU support. Often wins structured data competitions.

⚙️ Key hyperparameters

n_estimators, max_depth, learning_rate, subsample, colsample_bytree, reg_alpha/lambda.

📌 Objective: L(θ) = Σ loss(yᵢ, ŷᵢ) + Σ Ω(fₖ) where Ω penalizes tree complexity
import xgboost as xgb
import numpy as np
from sklearn.model_selection import train_test_split
from sklearn.metrics import roc_auc_score, classification_report

# Synthetic fraud data (imbalanced)
np.random.seed(42)
X_syn = np.random.randn(5000, 10)
y_syn = (X_syn[:,0] + X_syn[:,1]*0.5 + np.random.randn(5000)*0.3 > 0).astype(int)
# Make imbalance: 5% fraud
fraud_idx = np.where(y_syn == 1)[0]
keep_idx = np.random.choice(fraud_idx, size=int(len(fraud_idx)*0.3), replace=False)
y_syn[fraud_idx] = 0
y_syn[keep_idx] = 1
print(f"Fraud ratio: {y_syn.mean():.3f}")

X_train, X_test, y_train, y_test = train_test_split(X_syn, y_syn, test_size=0.2, random_state=42, stratify=y_syn)

# XGBoost basic
xgb_model = xgb.XGBClassifier(n_estimators=100, learning_rate=0.1, max_depth=4, random_state=42, use_label_encoder=False, eval_metric='logloss')
xgb_model.fit(X_train, y_train)
y_pred_xgb = xgb_model.predict(X_test)
y_proba_xgb = xgb_model.predict_proba(X_test)[:,1]
print(f"XGBoost AUC: {roc_auc_score(y_test, y_proba_xgb):.4f}")
🗳️ 2. Voting Classifier & Stacking Ensemble
🤝 Combine multiple algorithms for better predictions
📊 Hard Voting

Majority vote across classifiers. Simple, robust.

📈 Soft Voting

Averages predicted probabilities. Often outperforms hard voting.

🏗️ Stacking

Meta-learner trains on base model predictions. Can capture complex patterns.

from sklearn.ensemble import VotingClassifier, StackingClassifier
from sklearn.linear_model import LogisticRegression
from sklearn.tree import DecisionTreeClassifier
from sklearn.svm import SVC
from sklearn.ensemble import RandomForestClassifier

# Base models
lr = LogisticRegression(random_state=42, max_iter=1000)
dt = DecisionTreeClassifier(max_depth=5, random_state=42)
rf = RandomForestClassifier(n_estimators=50, random_state=42)

# Voting Classifier (soft)
voting_clf = VotingClassifier(estimators=[('lr', lr), ('dt', dt), ('rf', rf)], voting='soft')
voting_clf.fit(X_train, y_train)
voting_proba = voting_clf.predict_proba(X_test)[:,1]
print(f"Voting (soft) AUC: {roc_auc_score(y_test, voting_proba):.4f}")

# Stacking with XGBoost as meta-learner
base_models = [('lr', lr), ('dt', dt), ('rf', rf)]
stacking_clf = StackingClassifier(estimators=base_models, final_estimator=xgb.XGBClassifier(n_estimators=50, use_label_encoder=False, eval_metric='logloss', random_state=42))
stacking_clf.fit(X_train, y_train)
stacking_proba = stacking_clf.predict_proba(X_test)[:,1]
print(f"Stacking AUC: {roc_auc_score(y_test, stacking_proba):.4f}")
💡 Ensemble insight: Stacking often gives 1-3% AUC lift over individual models. Voting is simpler and less prone to overfitting.
⚙️ 3. GridSearchCV – Hyperparameter Tuning for XGBoost
🔍 Exhaustive search with cross-validation
📐 What is GridSearchCV?

Searches all combinations of hyperparameters, evaluates using k-fold CV, returns best estimator.

🎯 Why tune?

Default parameters rarely optimal. Tuning can improve AUC by 2-5%.

⚠️ Caution

Time-consuming for large grids. Use RandomizedSearchCV for large spaces.

from sklearn.model_selection import GridSearchCV

# Define parameter grid (reduced for speed)
param_grid = {
    'n_estimators': [50, 100],
    'max_depth': [3, 5],
    'learning_rate': [0.05, 0.1],
    'subsample': [0.8, 1.0]
}

xgb_base = xgb.XGBClassifier(random_state=42, use_label_encoder=False, eval_metric='logloss')
grid_search = GridSearchCV(xgb_base, param_grid, cv=3, scoring='roc_auc', n_jobs=-1, verbose=1)
grid_search.fit(X_train, y_train)

print(f"Best parameters: {grid_search.best_params_}")
print(f"Best CV AUC: {grid_search.best_score_:.4f}")

# Evaluate best model
best_xgb = grid_search.best_estimator_
y_proba_best = best_xgb.predict_proba(X_test)[:,1]
print(f"Test AUC with tuned XGBoost: {roc_auc_score(y_test, y_proba_best):.4f}")
💳 Credit Card Fraud Detection – Complete Pipeline
🏦 Real-world: Detect fraudulent transactions with ensemble & tuning
📊 Dataset simulation

Features: transaction amount, time, V1-V28 (PCA components). Highly imbalanced (~0.17% fraud).

🎯 Business impact

Catch fraud while minimizing false alarms. AUC and Precision-Recall critical.

🔧 Techniques used

XGBoost with scale_pos_weight (handles imbalance), Voting Ensemble, GridSearchCV.

# Simulated credit card fraud dataset (realistic structure)
np.random.seed(42)
n_samples = 10000
n_features = 20

X_fraud = np.random.randn(n_samples, n_features)
# Make fraud pattern: some features have different distribution
fraud_indices = np.random.choice(n_samples, size=int(n_samples*0.005), replace=False)  # 0.5% fraud
y_fraud = np.zeros(n_samples, dtype=int)
y_fraud[fraud_indices] = 1
# Alter features for fraud cases (make them distinguishable)
for idx in fraud_indices:
    X_fraud[idx, 2] += np.random.randn() * 2.5
    X_fraud[idx, 5] += np.random.randn() * 2.0

X_train_f, X_test_f, y_train_f, y_test_f = train_test_split(X_fraud, y_fraud, test_size=0.3, random_state=42, stratify=y_fraud)
print(f"Fraud % in train: {y_train_f.mean()*100:.2f}%")

# XGBoost with scale_pos_weight for imbalance
fraud_ratio = y_train_f.mean()
scale_pos_weight = (1 - fraud_ratio) / fraud_ratio
xgb_fraud = xgb.XGBClassifier(n_estimators=100, max_depth=4, learning_rate=0.1, 
                              scale_pos_weight=scale_pos_weight, random_state=42,
                              use_label_encoder=False, eval_metric='logloss')
xgb_fraud.fit(X_train_f, y_train_f)
y_proba_fraud = xgb_fraud.predict_proba(X_test_f)[:,1]
print(f"XGBoost Fraud AUC: {roc_auc_score(y_test_f, y_proba_fraud):.4f}")

# Feature importance
importance = xgb_fraud.feature_importances_
top_features = np.argsort(importance)[-5:][::-1]
print(f"Top 5 important features for fraud: {top_features}")
⚠️ Fraud detection insight: XGBoost with scale_pos_weight handles class imbalance effectively. Ensemble methods (Voting/Stacking) provide additional robustness against concept drift.

🛑 Early Stopping & Validation Set
⚙️ What is early stopping?

Halts training when validation metric stops improving. Prevents overfitting, saves time.

🎯 How to implement

Pass eval_set and early_stopping_rounds to XGBoost.

# Early stopping example
X_train_fs, X_val, y_train_fs, y_val = train_test_split(X_train_f, y_train_f, test_size=0.2, random_state=42)
xgb_early = xgb.XGBClassifier(n_estimators=500, learning_rate=0.05, max_depth=4, random_state=42,
                               use_label_encoder=False, eval_metric='logloss', early_stopping_rounds=20)
xgb_early.fit(X_train_fs, y_train_fs, eval_set=[(X_val, y_val)], verbose=False)
print(f"Best iteration: {xgb_early.best_iteration}")
print(f"Validation AUC: {xgb_early.best_score:.4f}")
