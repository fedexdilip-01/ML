📝 Day 12 – NLP Basics: Text Classification

Bag-of-Words, TF-IDF, Tokenization | SMS Spam Detection with Naive Bayes
📚
Text Representation

Bag-of-Words (BoW)

What? Represents text as word count vectors. Ignores order, only frequency.

Why? Simple, interpretable, effective for classification.

Where? Spam filtering, sentiment analysis, topic modeling.

⚖️
Weighted Frequency

TF-IDF

What? Term Frequency × Inverse Document Frequency. Downweights common words.

Why? Highlights discriminative words ("free", "win" for spam).

Where? Information retrieval, document similarity, text classification.

✂️
Text Preparation

Tokenization

What? Splits text into tokens (words, subwords, characters).

Why? First step in any NLP pipeline. Handles punctuation, case, stopwords.

Where? All text preprocessing pipelines.

✂️ 1. Tokenization – Breaking Text into Tokens
🔤 From raw text to token list
📐 Simple tokenization

Split by whitespace + punctuation removal. "Hello, world!" → ["Hello", "world"]

🧠 Advanced tokenization

NLTK, spaCy, or subword tokenizers (BPE) for OOV words.

⚙️ Preprocessing steps

Lowercasing, stopword removal, stemming/lemmatization.

import re
import nltk
from nltk.corpus import stopwords
from nltk.stem import PorterStemmer

# Download required NLTK data (run once)
# nltk.download('stopwords')

def tokenize_and_clean(text):
    # Lowercase
    text = text.lower()
    # Remove punctuation and digits
    text = re.sub(r'[^\w\s]', '', text)
    text = re.sub(r'\d+', '', text)
    # Split into words
    tokens = text.split()
    # Remove stopwords
    stop_words = set(stopwords.words('english'))
    tokens = [word for word in tokens if word not in stop_words]
    # Stemming
    stemmer = PorterStemmer()
    tokens = [stemmer.stem(word) for word in tokens]
    return tokens

sample = "WINNER!! Free entry to our $1000 lottery! Call now!"
print("Original:", sample)
print("Tokens:", tokenize_and_clean(sample))
# Output: ['winner', 'free', 'entri', 'lotteri', 'call']
💡 Pro tip: For spam detection, keep punctuation like '!!!' and '$' – they're strong spam indicators!
📚 2. Bag-of-Words – Text to Numeric Vector
📊 Vocabulary creation + frequency counting
📐 How it works

1) Build vocabulary from all documents. 2) Create vector of word counts for each document.

🎯 Example

["free money", "free call"] → vocab {free, money, call} → vectors [1,1,0] and [1,0,1]

⚙️ Limitations

Loses word order, high dimensionality, common words dominate.

BoW creates vocabulary → each document becomes frequency vector over vocabulary

from sklearn.feature_extraction.text import CountVectorizer

corpus = [
    "free money call now",
    "win free money",
    "call me back",
    "urgent free lottery win"
]

vectorizer = CountVectorizer()
X_bow = vectorizer.fit_transform(corpus)
print("Vocabulary:", vectorizer.get_feature_names_out())
print("\nBoW Matrix (dense):")
print(X_bow.toarray())
# Each row = document, each column = word frequency
⚖️ 3. TF-IDF – Smarter Word Weighting
📐 TF-IDF = Term Frequency × Inverse Document Frequency
📊 TF (Term Frequency)

How often word appears in document. More frequent = more important for this doc.

📈 IDF (Inverse Document Frequency)

log(N / df). Rare words get higher weight. Common words ("the", "a") get low weight.

🎯 Why better than BoW?

Downweights stopwords, highlights discriminative terms like "lottery", "winner" for spam.

📌 TF-IDF(t,d) = TF(t,d) × log(N / DF(t))
from sklearn.feature_extraction.text import TfidfVectorizer

tfidf_vectorizer = TfidfVectorizer()
X_tfidf = tfidf_vectorizer.fit_transform(corpus)

print("Feature names:", tfidf_vectorizer.get_feature_names_out())
print("\nTF-IDF Matrix:")
print(X_tfidf.toarray())
print("\n'free' appears in 3 documents → lower weight than 'urgent' (rare)")
📱 SMS Spam Detection – Complete Pipeline
📨 5,574 SMS messages (ham/spam) + Multinomial Naive Bayes
📊 Dataset

SMS Spam Collection: 4,827 ham (normal), 747 spam. Real-world imbalanced.

🎯 Model

Multinomial Naive Bayes – perfect for discrete word counts/TF-IDF.

⚙️ Pipeline

Text cleaning → TF-IDF vectorization → Naive Bayes → Evaluate.

import pandas as pd
import numpy as np
from sklearn.model_selection import train_test_split
from sklearn.feature_extraction.text import TfidfVectorizer
from sklearn.naive_bayes import MultinomialNB
from sklearn.pipeline import Pipeline
from sklearn.metrics import classification_report, confusion_matrix, accuracy_score
import matplotlib.pyplot as plt
import seaborn as sns

# Load SMS dataset (use built-in or create realistic sample)
# For demo, create realistic SMS data
np.random.seed(42)
n_samples = 5000
ham_messages = [
    "Hey, are we still meeting for lunch?",
    "Thanks for the update, appreciate it",
    "See you tomorrow at 5pm",
    "Can you pick up some milk on your way home?",
    "The project deadline has been extended",
    "Happy birthday! Hope you have a great day",
    "Don't forget to bring the documents",
    "I'll call you later tonight",
    "Meeting moved to 3pm in conference room A",
    "Thanks for your help with the presentation"
]
spam_messages = [
    "WINNER!! You've won a $1000 gift card! Click here",
    "URGENT: Your account has been suspended. Verify now",
    "FREE entry to our lottery, claim your prize",
    "CONGRATULATIONS! You're our lucky winner",
    "Lowest rates on loans, reply YES to apply",
    "You have been selected for a free iPhone",
    "Act now! Limited time offer",
    "Claim your $500 Walmart gift card",
    "Your verification code is 123456",
    "Exclusive deal just for you, click link"
]

# Generate dataset
ham_texts = np.random.choice(ham_messages, size=int(n_samples*0.87), replace=True)
spam_texts = np.random.choice(spam_messages, size=int(n_samples*0.13), replace=True)
texts = list(ham_texts) + list(spam_texts)
labels = ['ham'] * len(ham_texts) + ['spam'] * len(spam_texts)

df = pd.DataFrame({'text': texts, 'label': labels})
print(f"Dataset size: {len(df)}")
print(df['label'].value_counts())

# Train/test split
X_train, X_test, y_train, y_test = train_test_split(df['text'], df['label'], 
                                                    test_size=0.2, random_state=42, stratify=df['label'])

# Build pipeline: TF-IDF + Naive Bayes
pipeline = Pipeline([
    ('tfidf', TfidfVectorizer(stop_words='english', max_features=5000, ngram_range=(1,2))),
    ('nb', MultinomialNB())
])

# Train
pipeline.fit(X_train, y_train)

# Predict
y_pred = pipeline.predict(X_test)

# Evaluate
accuracy = accuracy_score(y_test, y_pred)
print(f"\n✅ Accuracy: {accuracy:.4f}")
print("\nClassification Report:")
print(classification_report(y_test, y_pred))

# Confusion Matrix
cm = confusion_matrix(y_test, y_pred)
plt.figure(figsize=(6,5))
sns.heatmap(cm, annot=True, fmt='d', cmap='Blues', xticklabels=['ham', 'spam'], yticklabels=['ham', 'spam'])
plt.xlabel('Predicted')
plt.ylabel('Actual')
plt.title('SMS Spam Detection - Confusion Matrix')
plt.show()

# Show example predictions
sample_messages = [
    "Hi honey, I'll be home soon",
    "CONGRATULATIONS! You won a free cruise!",
    "Reminder: dentist appointment tomorrow at 10am",
    "URGENT: Your account will be closed. Call now!"
]
for msg in sample_messages:
    pred = pipeline.predict([msg])[0]
    proba = pipeline.predict_proba([msg])[0]
    print(f"\nMessage: '{msg}'")
    print(f"Prediction: {pred} (spam prob: {proba[1]:.3f})")
📊 Expected results: Naive Bayes + TF-IDF achieves 97-99% accuracy on SMS spam. Common false positives: normal messages with 'free' or 'win' in legitimate contexts.

📊 Visualize top features

Extract feature names and Naive Bayes log probabilities.

# Get feature names and model coefficients
tfidf = pipeline.named_steps['tfidf']
nb = pipeline.named_steps['nb']
feature_names = tfidf.get_feature_names_out()

# Get log probabilities for spam class (class 1)
spam_log_probs = nb.feature_log_prob_[1]
ham_log_probs = nb.feature_log_prob_[0]

# Calculate ratio (spam vs ham)
spam_ratio = np.exp(spam_log_probs - ham_log_probs)
top_spam_idx = np.argsort(spam_ratio)[-20:][::-1]

print("Top 20 words most indicative of SPAM:")
for idx in top_spam_idx:
    print(f"  {feature_names[idx]}: {spam_ratio[idx]:.2f}x more likely in spam")

# Visualize

top_ratios = [spam_ratio[i] for i in top_spam_idx[:10]]

plt.figure(figsize=(10,5))

plt.show()

⚖️ Handling Class Imbalance in Spam Detection
📐 The problem

Only 13% spam in typical SMS datasets. Model might bias toward ham.

🎯 Solutions

Use class_weight='balanced', oversample spam (SMOTE), or adjust decision threshold.

# Naive Bayes with class weighting
